## 8.1 Introduction

Automating bivariate analyses (both statistics and charts) builds directly on the univariate automation techniques you learned previously. The key difference is that bivariate analysis focuses on _relationships_ between variables rather than the distribution of a single variable. Instead of asking “What does this variable look like?”, bivariate analysis asks “How does this variable relate to the outcome?”

In bivariate analysis, one variable typically plays the role of the _label_ (also called the _target_ or _outcome_), while the other is treated as a feature. Identifying the label correctly is essential, because it determines which analyses are meaningful and helps limit the number of unnecessary comparisons generated by automated functions.

In this book, bivariate statistics and visualizations are categorized based on the data type of the feature and the data type of the label. This allows you to treat bivariate analysis as a decision problem: once you know the data types involved, the appropriate statistical test and chart follow logically.

Specifically, there are four common bivariate relationship types: numeric–numeric, categorical–categorical, numeric–categorical, and categorical–numeric. A well-designed bivariate automation function uses this structure to determine whether to calculate Pearson’s _r_, a chi-square (_χ2_) test of independence, or an ANOVA (_F_-statistic), and to select an appropriate visualization for each case.

### Numeric-to-Numeric

We will start with numeric-to-numeric (N2N) relationships. When both the feature and the label are numeric, the appropriate primary statistic is the Pearson correlation coefficient (_r_). However, it is often useful to also calculate the regression line and the corresponding p-value, so we will compute all three.

Later in the course, we will extend this logic to include Kendall’s tau (τ) and Spearman’s rho (ρ) for cases where the assumptions of linear correlation are violated.

```python
def bivariate_stats(df, label, roundto=4):
  import pandas as pd
  from scipy import stats

  output_df = pd.DataFrame(columns=['p', 'r', 'y = m(x) + b'])

  for feature in df.columns:
    if feature != label:
      if pd.api.types.is_numeric_dtype(df[feature]):
        m, b, r, p, err = stats.linregress(df[feature], df[label])
        output_df.loc[feature] = [round(p, roundto), round(r, roundto), f'y = {round(m, roundto)}(x) + {round(b, roundto)}']

  return output_df
```

In this initial version of the function, we implemented several automation steps at once:

1. Defined the function and its inputs
1. Imported required packages
1. Created an output DataFrame to store results
1. Iterated through all features except the label
1. Applied a decision rule to isolate numeric features
1. Computed the appropriate statistics for N2N relationships

### Numeric-to-Numeric (N2N): Scatterplots

Let’s begin by creating a function that generates a scatterplot with a trendline. We could do this using either matplotlib or seaborn. If we used matplotlib, we would typically have to create the scatterplot and trendline separately and layer them together. But seaborn can create both in a single plot using _regplot_. At a minimum, _regplot_ needs a dataset (or two vectors) and the names of the feature and label. We will also embed key statistics (regression equation, r, r2, p, and n) directly in the chart so that you do not have to cross-reference the statistics table to interpret the relationship. Finally, we will make the function more reliable by handling missing data inside the function.

```python
def scatterplot(df, feature, label, roundto=3, linecolor='darkorange', title=None, savepath=None, show=True):
  import pandas as pd
  from matplotlib import pyplot as plt
  import seaborn as sns
  from scipy import stats

  if feature == label:
    raise ValueError('feature and label must be different.')

  # Handle missing data
  df_temp = df[[feature, label]].dropna()
  if df_temp.shape[0] < 3:
    raise ValueError('Not enough non-missing rows to plot and compute regression.')

  ax = plt.gca()

  # Create the plot (use data= signature for clarity and consistency)
  sns.regplot(data=df_temp, x=feature, y=label, ax=ax, line_kws={'color': linecolor})

  # Calculate regression statistics
  m, b, r, p, err = stats.linregress(df_temp[feature], df_temp[label])

  # Build annotation text
  textstr  = 'Regression line:' + '\n'
  textstr += 'y = ' + str(round(m, roundto)) + 'x + ' + str(round(b, roundto)) + '\n'
  textstr += 'r = ' + str(round(r, roundto)) + '\n'
  textstr += 'r2 = ' + str(round(r**2, roundto)) + '\n'
  textstr += 'p = ' + str(round(p, roundto)) + '\n'
  textstr += 'n = ' + str(df_temp.shape[0])

  # Optional title
  if title is None:
    title = str(feature) + ' vs. ' + str(label)
  ax.set_title(title)

  # Anchor text to axes so it stays stable across figure sizes
  ax.text(1.02, 0.02, textstr, fontsize=11, transform=ax.transAxes, va='bottom')

  # Optional save/show
  if savepath is not None:
    plt.savefig(savepath, bbox_inches='tight', dpi=200)
  if show:
    plt.show()

  return ax
```

Why not simply use _regplot_ directly? Because it can be useful to calculate the same statistics from the prior section and embed them in the visualization. That way, you can interpret strength, direction, and statistical significance in the same place you view the chart.

### Categorical-to-Numeric (C2N/N2C): Bar charts

There are many options in seaborn for C2N/N2C relationships. The most basic and straightforward is a bar chart using _barplot_, so we will use that. Your goal is to create a function similar to the one above that embeds a one-way ANOVA F statistic and p-value in the chart. If you have trouble, examine the code below for help. In addition, we will make the function more robust by (1) handling missing data inside the function and (2) automatically swapping inputs so that the categorical variable always appears on the x-axis.

```python
def bar_chart(df, feature, label, roundto=3, title=None, savepath=None, show=True):
  import pandas as pd
  from scipy import stats
  from matplotlib import pyplot as plt
  import seaborn as sns

  if feature == label:
    raise ValueError('feature and label must be different.')

  # Ensure we plot categorical on x-axis and numeric on y-axis
  is_num_feature = pd.api.types.is_numeric_dtype(df[feature])
  is_num_label = pd.api.types.is_numeric_dtype(df[label])

  if is_num_feature and not is_num_label:
    num = feature
    cat = label
  elif not is_num_feature and is_num_label:
    num = label
    cat = feature
  else:
    raise ValueError('bar_chart requires one numeric and one categorical variable.')

  # Handle missing data
  df_temp = df[[cat, num]].dropna()
  if df_temp[cat].nunique() < 2:
    raise ValueError('Not enough categories to compare (need at least 2).')

  ax = plt.gca()

  # Create the plot (modern seaborn signature)
  sns.barplot(data=df_temp, x=cat, y=num, ax=ax, errorbar=None)

  # Create the group lists needed to calculate one-way ANOVA
  groups = list(df_temp[cat].unique())
  group_lists = [df_temp[df_temp[cat] == g][num] for g in groups]

  results = stats.f_oneway(*group_lists)
  F = results[0]
  p_anova = results[1]

  # Pairwise t-tests with Bonferroni correction
  ttests = []
  for i1, g1 in enumerate(groups):
    for i2, g2 in enumerate(groups):
      if i2 > i1:
        vals_1 = df_temp[df_temp[cat] == g1][num]
        vals_2 = df_temp[df_temp[cat] == g2][num]
        t, p = stats.ttest_ind(vals_1, vals_2, equal_var=False, nan_policy='omit')
        ttests.append([str(g1) + ' - ' + str(g2), round(t, roundto), round(p, roundto)])

  p_threshold = 0.05 / max(1, len(ttests))

  # Annotation text
  textstr  = 'ANOVA' + '\n'
  textstr += 'F: ' + str(round(F, roundto)) + '\n'
  textstr += 'p: ' + str(round(p_anova, roundto)) + '\n'
  textstr += 'n: ' + str(df_temp.shape[0]) + '\n\n'

  for ttest in ttests:
    if ttest[2] <= p_threshold:
      if 'Sig. comparisons (Bonferroni)' not in textstr:
        textstr += 'Sig. comparisons (Bonferroni)' + '\n'
      textstr += str(ttest[0]) + ': t=' + str(ttest[1]) + ', p=' + str(ttest[2]) + '\n'

  if title is None:
    title = str(num) + ' by ' + str(cat)
  ax.set_title(title)

  ax.text(1.02, 0.02, textstr, fontsize=10.5, transform=ax.transAxes, va='bottom')

  if savepath is not None:
    plt.savefig(savepath, bbox_inches='tight', dpi=200)
  if show:
    plt.show()

  return ax
```

In the _bar_chart()_ function above, we created the visualization and embedded a one-way ANOVA F statistic and p-value. We also computed pairwise t-tests and used a Bonferroni-corrected threshold to reduce false positives when comparing multiple groups. Finally, the function automatically ensures the categorical variable is plotted on the x-axis, which makes the chart consistent even if the user passes the arguments in the opposite order.

One way to improve this function would be to bin rare categorical values (for example, values representing less than 5% of the dataset) into a category called _Other_. This is useful because it reduces visual clutter and reduces the number of t-tests for features with many values. We will come back to this idea later.

### Categorical-to-Categorical (C2C): Heat maps

For categorical-to-categorical relationships, we start by building a crosstab table of observed counts. Then we use that crosstab to compute a chi-square (X2) test. Finally, we visualize the observed counts as a heat map. In this function, we also handle missing data and (optionally) bin rare categories into _Other_ to keep the heat map readable.

```python
def bin_categories(df, feature, cutoff=0.05, replace_with='Other'):
  import pandas as pd

  # Work on a copy to avoid mutating the caller's DataFrame unexpectedly
  df_out = df.copy()

  # Identify rare categories based on proportion (ignore missing values)
  proportions = df_out[feature].value_counts(dropna=True) / df_out[feature].dropna().shape[0]
  other_list = proportions[proportions < cutoff].index

  # Replace values in rare categories with 'Other'
  df_out.loc[df_out[feature].isin(other_list), feature] = replace_with

  return df_out
```

```python
def crosstab(df, feature, label, roundto=3, cutoff=0.05, title=None, savepath=None, show=True):
  import pandas as pd
  from scipy.stats import chi2_contingency
  from matplotlib import pyplot as plt
  import seaborn as sns

  if feature == label:
    raise ValueError('feature and label must be different.')

  # Handle missing data
  df_temp = df[[feature, label]].dropna()
  if df_temp.shape[0] == 0:
    raise ValueError('No non-missing rows available for this pair of variables.')

  # Bin rare categories for readability (applies to feature by default)
  df_temp = bin_categories(df_temp, feature, cutoff=cutoff)

  # Observed counts (what we typically want to visualize)
  ct = pd.crosstab(df_temp[feature], df_temp[label])

  # Chi-square based on observed counts
  X2, p, dof, expected = chi2_contingency(ct)

  ax = plt.gca()
  sns.heatmap(ct, annot=True, fmt='d', ax=ax)

  if title is None:
    title = str(feature) + ' vs. ' + str(label)
  ax.set_title(title)

  textstr  = 'X2: ' + str(round(X2, roundto)) + '\n'
  textstr += 'p: ' + str(round(p, roundto)) + '\n'
  textstr += 'dof: ' + str(dof) + '\n'
  textstr += 'n: ' + str(df_temp.shape[0])

  ax.text(1.02, 0.02, textstr, fontsize=10.5, transform=ax.transAxes, va='bottom')

  if savepath is not None:
    plt.savefig(savepath, bbox_inches='tight', dpi=200)
  if show:
    plt.show()

  return ax
```

## 8.4 Controller Function

Now that we have a set of visualization helper functions to go with our bivariate statistics logic, we can combine everything into a single _controller_ function. The controller’s job is simple: iterate through each feature, identify the relationship type (N2N, C2N/N2C, or C2C), compute the correct statistics, and then call the matching visualization function. To do this, we will copy the latest version of the bivariate statistics logic and integrate the visualization function calls into the appropriate branches of the decision logic.

```python
def bivariate(df, label, roundto=4, viz=True):
  import pandas as pd
  from scipy import stats

  output_df = pd.DataFrame(
    columns=['missing', 'p', 'r', 'τ', 'ρ', 'y = m(x) + b', 'F', 'X2', 'skew', 'unique', 'values']
  )

  for feature in df.columns:
    if feature != label:
      # Work with a temporary 2-column slice so missing-data handling is consistent for every relationship
      df_temp = df[[feature, label]].dropna()

      missing = (df.shape[0] - df_temp.shape[0]) / df.shape[0]
      unique = df_temp[feature].nunique()

      # Bin categories on the TEMP slice (not the full df) to avoid mutating the dataset globally
      if not pd.api.types.is_numeric_dtype(df_temp[feature]):
        df_temp = bin_categories(df_temp, feature)

      if not pd.api.types.is_numeric_dtype(df_temp[label]):
        df_temp = bin_categories(df_temp, label)

      # N2N: numeric label and numeric feature
      if pd.api.types.is_numeric_dtype(df_temp[feature]) and pd.api.types.is_numeric_dtype(df_temp[label]):
        m, b, r, p, err = stats.linregress(df_temp[feature], df_temp[label])
        tau, tp = stats.kendalltau(df_temp[feature], df_temp[label])
        rho, rp = stats.spearmanr(df_temp[feature], df_temp[label])

        output_df.loc[feature] = [
          f'{missing:.2%}', round(p, roundto), round(r, roundto), round(tau, roundto),
          round(rho, roundto), f'y = {round(m, roundto)}(x) + {round(b, roundto)}',
          '-', '-', df_temp[feature].skew(), unique, '-'
        ]

        if viz:
          scatterplot(df_temp, feature, label, roundto)  # Call the scatterplot function

      # C2C: categorical label and categorical feature
      elif (not pd.api.types.is_numeric_dtype(df_temp[feature])) and (not pd.api.types.is_numeric_dtype(df_temp[label])):
        contingency_table = pd.crosstab(df_temp[feature], df_temp[label])
        X2, p, dof, expected = stats.chi2_contingency(contingency_table)

        output_df.loc[feature] = [
          f'{missing:.2%}', round(p, roundto), '-', '-', '-', '-', '-', round(X2, roundto),
          '-', unique, df_temp[feature].unique()
        ]

        if viz:
          crosstab(df_temp, feature, label, roundto)  # Call the crosstab function

      # C2N / N2C: one numeric, one categorical
      else:
        if pd.api.types.is_numeric_dtype(df_temp[feature]):
          skew = df_temp[feature].skew()
          num = feature
          cat = label
        else:
          skew = '-'
          num = label
          cat = feature

        groups = df_temp[cat].unique()
        group_lists = []
        for g in groups:
          g_list = df_temp[df_temp[cat] == g][num]
          group_lists.append(g_list)

        results = stats.f_oneway(*group_lists)
        F = results[0]
        p = results[1]

        output_df.loc[feature] = [
          f'{missing:.2%}', round(p, roundto), '-', '-', '-', '-', round(F, roundto), '-', skew,
          unique, df_temp[cat].unique()
        ]

        if viz:
          bar_chart(df_temp, cat, num, roundto)  # Call the bar chart function

  return output_df.sort_values(by=['p'])
```

Notice what the controller function accomplishes: it applies the same missing-data handling for every feature, uses a simple decision tree to determine the relationship type, records the appropriate statistics in a single output table, and then calls the matching visualization helper function (scatterplot, bar chart, or heat map).
